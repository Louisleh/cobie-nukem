# Visual art brief contract

Use one brief per bounded production target.

```yaml
work_id:
target:
baseline_revision:
canonical_views:
gameplay_readability:
silhouette_goal:
visual_story:
palette:
value_hierarchy:
materials:
lighting_and_fog:
animation_or_state_vocabulary:
platform_budgets:
source_and_license:
owned_paths:
preserved_gameplay_contracts:
acceptance_evidence:
human_review_questions:
out_of_scope:
```

The brief is incomplete when it contains only mood adjectives. It must say what the player needs to notice first, what can remain quiet, which gameplay shapes cannot move, and how the Web/iPad version degrades.

For enemy sprites, include world-space height, orthographic camera, feet baseline, eight-direction order, frame dimensions, atlas layout, alpha treatment, reaction orientation, distant animation rate, and collision silhouette.

For materials, include texel density, tiling scale, albedo range, roughness range, normal strength, packed-map channels, mipmap/filter settings, and whether markings/puddles/unique dirt are separate decals.

For touch/UI, include safe-area insets, minimum target size, information priority, handedness, pressed/held/cooldown/focus states, and the four supported aspect families.
